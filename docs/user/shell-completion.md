# Shell completion

`sentinel` ships first-class completion for the four most common shells.
Run it once and your shell will tab-complete commands, options, and
known values (module names, severities, run IDs).

## Install

| Shell        | Command                                    |
| ------------ | ------------------------------------------ |
| `bash`       | `sentinel --install-completion bash`       |
| `zsh`        | `sentinel --install-completion zsh`        |
| `fish`       | `sentinel --install-completion fish`       |
| `powershell` | `sentinel --install-completion powershell` |

Re-open your terminal — or source the appropriate rc file — and start
typing:

```bash
$ sentinel <Tab>
audit       chaos      doctor     init       perf       security
api         ci         fix        llm-audit  plugins    test
a11y        compliance functional mcp        report     visual

$ sentinel audit --<Tab>
--url           --config        --json          --fail-under    --dry-run
--module        --quiet         --ci            --verbose       --watch
```

## What gets completed

- **Command names** — every `sentinel` subcommand.
- **Option names** — every flag on every subcommand.
- **Known values** — module names (`functional`, `security`, …), output
  formats (`html`, `json`, `sarif`, `junit`, `md`), and severities
  (`critical`, `high`, `medium`, `low`, `info`).
- **Paths** — `--config`, `--out-dir`, and other path-valued options.
- **Recent run IDs** — `sentinel report --run-id <Tab>` lists the
  most recent runs under `.sentinel/runs/`.

## Removing

```bash
# bash
rm "${BASH_COMPLETION_USER_DIR:-${XDG_DATA_HOME:-$HOME/.local/share}/bash-completion}/completions/sentinel"

# zsh
rm -f "${fpath[1]}/_sentinel"
# (and remove `autoload -U compinit && compinit` from .zshrc if you added it)

# fish
rm "$HOME/.config/fish/completions/sentinel.fish"

# powershell
# delete the line that loads the completion in your $PROFILE
```

## How it works

Completion is generated by Typer at install time and lives in the
shell's standard per-user completion directory. The CLI itself does
not need to be running for completion to work — your shell loads the
generated script.

If you upgrade SentinelQA and a command name changes, re-run
`sentinel --install-completion <shell>` to refresh the script.
