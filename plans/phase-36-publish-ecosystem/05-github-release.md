# Task 36.05 — GitHub Release workflow

## Deliverables

- `.github/workflows/github-release.yml`:
  - Trigger: `on: { push: { tags: ["v*"] } }`.
  - Permission `contents: write`.
  - Uses `softprops/action-gh-release@v2`:
    - `name: <tag>`
    - `body_path: <generated release notes from CHANGELOG.md tag
      section>` (extractor script
      `scripts/release/extract_release_notes.py` slices the
      `CHANGELOG.md` section between `## [<tag>]` and the next `## [`).
    - `files: dist/*.whl,dist/*.tar.gz,dist/sentinelqa-ts-runtime-*.tgz,
      dist/sbom-*.json` (the SBOM is generated by the Phase 33
      module against the built artifacts before this step runs).
    - `make_latest: legacy` — the latest stable tag overrides
      `latest`.
    - `discussion_category_name: Releases` (owner enables
      Discussions; documented).
- `scripts/release/extract_release_notes.py` parses `CHANGELOG.md`,
  extracts the requested tag's section, normalises Markdown for
  GitHub display (turns relative links into absolute repo-URL links).

## Tests required

- `tests/integration/release/test_extract_release_notes.py` —
  fixtures of CHANGELOG.md sections; extractor returns the expected
  Markdown for each tag.

## Definition of Done

- [ ] Workflow committed.
- [ ] Extractor produces clean release notes.
- [ ] `STATUS.md` updated.
